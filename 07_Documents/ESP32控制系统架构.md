# ESP32 控制系统架构

## 1. 系统角色定位

ESP32 作为本项目的中央控制单元，负责整合运动控制、传感器采集和决策逻辑。其优势包括丰富 GPIO、低功耗特性和 Arduino 框架支持。

## 2. 软件架构层次

### 2.1 主控层

- `src/main.cpp`
- 系统初始化：串口、模块初始化
- 主循环：周期调用控制逻辑

### 2.2 模块层

- `motor` 模块：实现步进电机基础驱动与移动接口
- `sensor` 模块：实现 HX711 力传感器和限位开关读取
- `control` 模块：实现夹爪闭环控制与 PID 预留

## 3. 功能模块说明

### 3.1 Motor 模块

- 初始化电机驱动
- 设置最大速度与加速度
- 提供位置移动与运行执行接口

### 3.2 Sensor 模块

- HX711 采集力值
- 限位开关检测夹爪开/闭边界
- 对外提供统一传感接口

### 3.3 Control 模块

- 读取传感器数据
- 控制步进电机动作
- 预留 PID 力控逻辑接口
- 实现基础的开合夹爪操作

## 4. 硬件接口规划

- 步进电机：STEP=18, DIR=19, EN=21
- HX711：DT=4, SCK=5
- 限位开关：OPEN=12, CLOSE=13

## 5. 未来扩展方向

- 集成 PID 或自适应力控算法
- 支持更复杂夹爪状态机
- 添加串口或无线通信诊断
- 扩展至多传感器融合与安全监测
